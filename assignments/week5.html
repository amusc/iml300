    • Website Critique: Layout

    Good Layout

    I chose this as an example of website that features a good layout (in line with our lecture.) The website is structured off a grid, is responsive, differentiates based on page and project. I think that the strategies it employs such as changing project images upon rollover keeps the website engaging yet minimal. I also think that text was integrated well, which is incredibly important not only visually but in regards to relaying information in a layout.

    Bad Layout

    I found this to be an example of a bad layout for a number of reasons. Though it does follow a grid within its layout, this grid is very static— the website is not responsive, and something about its symmetry and proportional sizing doesn’t seem to work exceptionally well. I think that the breakdown of information within the grid, such as the multiple navigation bars (especially after exploring had been placed in them, and the content of those pages) is visually confusing. The page is text/information dense, but this information could all be relayed in a similar grid structure much more succinctly (such as how margins, alignment, and other aspects have been created.)
